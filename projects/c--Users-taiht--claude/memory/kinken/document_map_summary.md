The KINKEN document set spans the full lifecycle:
- project direction and governance
- UI/product definition
- data collection platform and ETL
- search platform and APIs
- schema/index design
- auth and integrations
- logging and analytics
- testing, migration, and release

Recommended reading order:
1. Inception / project direction / working agreement
2. Architecture and data collection overview
3. Search API, filters, schema, and index design
4. Auth and non-functional requirements
5. Logging, GA, testing, migration, and release

This map is useful because KINKEN is not just a UI project; it is also a search platform, data pipeline, and integration hub.
