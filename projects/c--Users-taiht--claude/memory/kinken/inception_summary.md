Key purpose of KINKEN:
- Centralize and standardize LIXIL metal-product information.
- Help users find needed information quickly, even without knowing where it lives.
- Improve internal work efficiency and connect smoothly to downstream work.
- Act as a shared product-information platform and reusable search API for other systems.

Two product viewpoints:
- End-user product: metal product information search for contractors, dealers, and internal users.
- Platform product: shared LIXIL product information platform for other systems and services.

Scope and priorities:
- Focus on MUST functions from existing KINKEN/SAGAS.
- Search quality should be at least current level, ideally above PoC expectation.
- NEED/WANT items are adjustable.
- Out of scope at inception included direct order-system integration, non-QA inquiry channels, and some advanced additions like LLM integration.

Stakeholder landscape:
- Business, digital, security, infra, design, end users.
- Connected systems and domains include PIM, QA, CAD, web catalog, repair/manual systems, and LIXIL-X.

Schedule shape:
- Requirements → design → development → system test → acceptance test → release → migration.
- Timeline spans 2025 to 2026.
